Physics Based Cable

 

Block Description

This block is a power transmission cable, with resistivity based on physical cable parameters. Developed in collaboration with OpenMDAO.

Mask Physical Parameters (can be calculated from iDesign)

Conductor Radius [mm] Radius of conductor, based on 100% copper cable.
Insulator Thickness [mm] Thickness of insulation on cable, based on standard PVC insulation.
Mass [g] Total estimated cable mass (does not impact any calculations).
Resistance at 20C [Ohm] Total cable resistance at reference temperature.

Input Variables

I_Demand The current in Amps that is being demanded of the cable block.
Vin The bus voltage that the cable block sees at one end.
Temp (Ra) Temperature of the cable in Rankine, can either be a constant ambient or updated by the Temperature Calculation block.
Catastrophic Fault Toggle Input for total component failure (Boolean, 0 or unconnected for regular operation)
Degradation Parameter Gradual degrading input for resistance (0 or unconnected for regular operation. Resistance is multiplied by any other value directly.)

Output Variables

I_demanded Current demanded from blocks providing voltage to this block.
VOut Output voltage provided to subsequent blocka.
Heat (W) Heat Generated

iDesign Mask Parameters

Max Voltage Input [V] Expected maximum voltage for that portion of the system.
Allowable Voltage Drop [%] Maximum voltage drop across the cable.
Max Current Output [A] Expected maximum current for that portion of the system.
Desired Cable Length [cm] Expected length of cabling, which affects resistivity and weight.
Factor of Safety Factor to increase cable sizing to accomodate unexpected power spikes and degredation.
